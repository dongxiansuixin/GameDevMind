# 🎮 命令模式（Command Pattern）— 游戏输入录制与回放

## 📖 对应图谱

> [设计模式](../../../../mds/1.基础能力/1.2.1.设计模式.md) → 行为型模式 → **命令模式**

## 💡 核心概念

**命令模式**将请求封装为对象，从而允许你使用不同的请求、队列或日志来参数化其他对象，同时支持可撤销的操作。

### 游戏场景应用

| 应用场景 | 说明 |
|----------|------|
| **输入录制** | 格斗游戏的连招录制、RTS 的操作序列记录 |
| **回放系统** | 存储玩家操作而非视频帧，实现极小体积的比赛回放 |
| **Undo/Redo** | 关卡编辑器中的撤销/重做操作历史 |
| **命令队列** | 网络同步中的输入缓冲、动作排队 |

## 🏗️ 架构设计

```
┌─────────────┐
│  PlayerInput │  玩家输入（键盘/手柄/触屏）
└──────┬──────┘
       │ 创建
       ▼
┌─────────────────────────────────┐
│         ICommand (接口)          │
│  + Execute(Player&)             │
│  + Undo(Player&)                │
│  + GetName()                    │
└────────┬───────────┬────────────┘
         │           │
    ┌────▼───┐  ┌───▼────┐  ┌──────────┐
    │ Move   │  │ Attack │  │  Skill   │
    │Command │  │Command │  │ Command  │
    └────────┘  └────────┘  └──────────┘
                    │
              ┌─────▼──────┐
              │CommandQueue│  命令队列
              │ Record()   │  录制
              │ ExecuteAll │  批量执行
              │ UndoLast() │  撤销
              │ RedoLast() │  重做
              │ Replay()   │  回放
              └────────────┘
```

## 📁 文件说明

| 文件 | 说明 |
|------|------|
| `command.hpp` | ICommand 基类 + MoveCommand / AttackCommand / SkillCommand |
| `command_queue.hpp` | 命令队列：录制、批量执行、撤销、重做、回放 |
| `main.cpp` | 演示：录制序列 → 执行 → 撤销 → 重做 → 完整回放 |
| `CMakeLists.txt` | CMake 构建配置（C++17） |

## 🎯 关键设计要点

1. **命令对象化**：每个操作都是独立对象，携带执行和撤销所需的所有信息
2. **状态快照**：SkillCommand 保存完整 Player 快照，确保撤销安全
3. **双端队列**：`history_`（执行历史）/ `undone_`（撤销栈）支持任意深度的 Undo/Redo
4. **录制与回放分离**：`recorded_names_` 用于回放，`pending_` 用于实时执行
