# 🎯 对象池模式（Object Pool）— 子弹/粒子系统性能优化

## 📖 对应图谱

> [设计模式](../../../../mds/1.基础能力/1.2.1.设计模式.md) → 创建型模式 → **对象池模式**

## 💡 核心概念

**对象池**预先分配一批对象，需要时从池中获取（Acquire），用完后归还（Release），避免频繁的 `new`/`delete` 操作。

### 游戏场景应用

| 应用场景 | 说明 |
|----------|------|
| **子弹系统** | 射击游戏每帧数十颗子弹，生命周期短，频繁创建销毁 |
| **粒子系统** | 爆炸、尾迹等粒子特效，峰值可达数百/帧 |
| **敌人/AI** | 波次刷怪，怪物死亡后回池而非销毁 |
| **音效播放** | 同时播放的音效数量有限，AudioSource 池化复用 |
| **网络包** | 服务器收发包缓冲区复用，减少 GC 压力 |

## 🏗️ 架构设计

```
┌──────────────────────────────────┐
│       ObjectPool<Bullet>         │
│                                  │
│  pool_: [B0][B1][B2]...[Bn]     │  ← 预分配对象数组
│  free_list_: [2, 5, 3, ...]     │  ← 空闲索引栈
│                                  │
│  Acquire() → 弹出索引，返回对象引用  │
│  Release() → 压入索引              │
│  Expand()  → 池耗尽时自动翻倍扩容    │
└──────────────────────────────────┘
          ▲              │
          │ Acquire      │ Release
          │              ▼
    ┌──────────┐   ┌──────────┐
    │  Game    │   │  Bullet  │
    │  Loop    │──▶│  Update  │──▶ lifetime≤0 → Release
    └──────────┘   └──────────┘
```

## 🎯 关键设计要点

1. **预分配策略**：初始化时分配 `initial_size` 个对象，覆盖常规需求
2. **自动扩容**：池耗尽时翻倍扩容，避免游戏卡顿（扩容只发生一次或几次）
3. **索引栈**：`free_list_` 用 `vector<size_t>` 实现 LIFO，O(1) 获取/归还
4. **Reset 机制**：使用 C++17 `if constexpr` 检测对象是否有 `Reset()` 方法，有则自动调用
5. **线程模型**：游戏主线程专用，不加锁，避免同步开销
6. **Release 安全**：`assert` 检测归还对象是否属于本池，防止野指针

## ⚖️ 对象池 vs new/delete 对比

| 维度 | new/delete | 对象池 |
|------|-----------|--------|
| 分配速度 | 慢（系统调用） | 快（O(1) 索引栈） |
| 内存碎片 | 严重（频繁分配不同大小） | 无（固定大小，预分配） |
| 缓存友好 | 差（对象分散在堆各处） | 好（连续内存） |
| 峰值内存 | 不可控 | 可控（池容量上限） |
| 实现复杂度 | 零 | 需要维护池状态 |

## 🔧 生产环境增强建议

- 对象内嵌池索引，加速 Release 时的查找（O(1) 代替 O(n)）
- 添加 `max_size` 上限，防止无限扩容
- 支持 `shared_ptr`/`weak_ptr` 包装，防止悬空引用
- Debug 模式下追踪 Acquire/Release 调用栈，检测泄漏
