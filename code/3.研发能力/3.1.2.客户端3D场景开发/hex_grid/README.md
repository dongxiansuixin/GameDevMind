# 六边形网格 + A* 寻路 完整 C++ 示例

## 概述

本项目是一个**纯头文件（header-only）**的 C++17 六边形网格与 A\* 寻路实现，包含完整的 ASCII 可视化演示。代码设计用于游戏开发中的地图系统参考，所有模块解耦清晰，可独立复用。

## 六边形网格 vs 四边形网格

| 特性 | 四边形网格 (Square) | 六边形网格 (Hex) |
|------|---------------------|-------------------|
| **邻居数量** | 4 (四连通) 或 8 (八连通) | 6 个，间距均匀 |
| **距离精度** | 对角线距离需修正 (√2) | 所有邻居等距，曼哈顿距离天然精确 |
| **移动方向** | 4/8 方向，对角线造成"加速"问题 | 6 方向均匀，无对角线优先问题 |
| **视觉自然感** | 街区感强 | 蜂巢状，更有机 |
| **覆盖效率** | 存在覆盖间隙问题 | 完美密铺平面 |
| **寻路复杂度** | 简单 | 坐标转换稍复杂 |

**六边形核心优势**：
1. **等距邻居** — 所有6个方向的移动代价均等，没有四边形中对角线"作弊"的问题
2. **距离计算精确** — Cube 坐标系下曼哈顿距离 `/2` 即为准确格数
3. **更自然的区域覆盖** — 圆形范围、视野、溅射范围在六边形网格上更接近真实几何形状
4. **策略游戏首选** — 《文明》系列、《英雄无敌》等经典策略游戏均采用六边形网格

## 坐标系统

本项目实现三种坐标表示，均可互相转换：

### 1. Cube 坐标 `(q, r, s)`
- 三维坐标，约束 `q + r + s = 0`
- 每个六边形可视为立方体在平面上的投影
- 优点：距离计算最简单，对称性最好
- 计算公式：`distance = max(|Δq|, |Δr|, |Δs|)`

### 2. Axial 坐标 `(q, r)`
- Cube 坐标去掉冗余的 `s` 分量（`s = -q - r`）
- 存储效率最高，最常用于实际代码

### 3. Offset 坐标 `(col, row)`
- 最直观的二维数组索引
- **odd-r**：奇数行右移半格（本项目采用）
- 适合直接映射到屏幕像素 / 数组存储

### 坐标选择建议
| 场景 | 推荐坐标 |
|------|----------|
| 寻路算法 & 距离计算 | Cube / Axial |
| 网格存储 & 序列化 | Offset |
| 邻居查询 | Axial |

转换函数在 `hex_coord.hpp` 中全量提供。

## A* 寻路算法原理

A\* 寻路是 Dijkstra 算法的启发式增强版，核心公式：

```
f(n) = g(n) + h(n)
```

- **g(n)**：从起点到节点 n 的**实际代价**（已行走）
- **h(n)**：从节点 n 到终点的**启发式估计代价**（预估剩余）
- **f(n)**：总评估值，决定探索优先级

### 算法步骤
1. 将起点加入 open 列表（优先队列）
2. 从 open 列表中取出 `f` 值最小的节点
3. 若该节点为终点 → 回溯路径，返回
4. 扩展该节点的所有邻居，更新 `g` 值
5. 对新节点计算 `f`，加入 open 列表
6. 重复 2-5，直到 open 列表为空

### 关键设计
- **启发函数 `h`** 必须**可接受**（admissible，不高估实际代价）才能保证最优解
- 本项目使用**六边形曼哈顿距离**作为启发函数，天然满足可接受性
- `astar.hpp` 是**完全模板化**的实现，不依赖任何具体坐标类型

## 运行方法

### 环境要求
- C++17 编译器（gcc 8+ / clang 7+ / MSVC 2019+）
- CMake 3.16+

### 编译运行

```bash
cd hex_grid/

# 方法一：CMake
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make
./hex_demo

# 方法二：直接编译（单文件）
g++ -std=c++17 -O2 -o hex_demo main.cpp && ./hex_demo
```

### 预期输出
- 彩色 ASCII 六边形网格地图（12×10）
- 地形用不同颜色和字符标识
- 红色 `●` 标记 A\* 寻路结果路径
- 路径详情：每步坐标、地形、代价、累计代价
- 坐标转换示例
