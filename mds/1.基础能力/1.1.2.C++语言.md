## 语言应用

| 维度 | 内容 |
|------|------|
| **为什么选择 C++ 开发游戏？** | C++ 够"低级"，允许控制内存、指令、硬件；运行效率高（掌握好、优化好）；主机硬件会针对 C++ 做优化；适合需要高度优化的场景（硬件限制、成本考量） |
| **游戏开发场景** | 引擎开发（运行时库需最大化效率）；服务器开发（发挥每台服务器效能）；工具开发（高性能需求工具） |

## 数据类型

| 维度 | 内容 |
|------|------|
| **作用** | 提供丰富的数据类型（基本类型、复合类型） |
| **应用场景** | 所有需要存储和操作数据的场景 |
| **常用类型** | 整数（int、long、long long）；浮点（float、double、long double）；字符（char、wchar_t）；布尔（bool）；复合类型（数组、结构体、类） |

| 问题 | 解决方向 |
|------|----------|
| **溢出问题**：整形运算超出 32 位，时间相关变量跑半个月可能溢出 | 注意运算顺序（如 `D = A / C * B`）；使用更大类型（long long）；使用 `std::numeric_limits` 溢出检查；时间变量用专门类型 |
| **精度问题**：小数位太多无法正确计算，Z 闪烁 | 根据精度选 float/double；图形编程考虑 double 或定点数；避免直接比较浮点数相等，用误差范围 |
| **有损截断**：高精度赋给低精度，数据被破坏性转化 | 使用 `narrow_cast` 等安全转换；使用 `static_cast` 显式转换；转换前检查范围；使用 `-Wnarrowing` 警告 |

| 要点和思考方向 |
|----------------|
| 根据数据范围选择整数类型，根据精度需求选择浮点类型 |
| 注意类型转换安全性，使用现代 C++ 的类型安全转换方法 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明数据范围、精度需求、平台（32/64 位），如「时间变量可能跑半个月溢出」「3D 坐标需高精度避免 Z-fighting」 |
| **方法** | 提供目标平台和 C++ 标准版本；要求 AI 使用 `std::numeric_limits`、`narrow_cast` 等安全手段 |
| **应用** | 分析溢出风险并建议类型；生成带范围检查的转换代码；解释 float/double 精度差异 |
| **提示词范例** | 「这段代码中 `int elapsed = end - start` 可能溢出，请改用合适类型并添加溢出检查，目标平台 64 位」 |

## 指针

| 维度 | 内容 |
|------|------|
| **说明** | 指针是存储地址的变量，指向内存中的一个存储单元 |
| **意义** | 只动结构不动内容，可只换门牌号而不用搬家 |

| 问题 | 解决方向 |
|------|----------|
| **野指针**：指向的对象已释放，指针仍存在，继续操作会出错 | 释放后置空（`ptr = nullptr`）；使用智能指针；避免返回局部变量指针；使用 RAII |
| **空指针**：使用空指针访问导致崩溃 | 使用前检查是否为空；使用引用替代（如可能）；`get()` 前检查 |
| **指针算术错误**：导致访问越界 | 注意指针类型和步进距离；使用数组索引；使用 STL 容器 |

| 要点和思考方向 |
|----------------|
| 指针释放后及时置空，优先使用智能指针 |
| 使用指针前检查是否为空，理解指针与引用的区别 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明指针用途、生命周期、是否可能为空；如「这个指针在函数返回后仍会被使用」「需要传递可选对象」 |
| **方法** | 明确所有权（谁负责释放）；要求 AI 优先考虑智能指针；提供调用方/被调用方上下文 |
| **应用** | 将裸指针重构为智能指针；检查空指针使用；生成 RAII 封装 |
| **提示词范例** | 「将这段使用裸指针的代码改为 unique_ptr，并确保释放后不会出现野指针」 |

## 指针与数组

| 维度 | 内容 |
|------|------|
| **作用** | 理解指针和数组的关系，正确使用指针访问数组 |
| **关系** | 数组名在大多数情况下退化为指向首元素的指针 |
| **sizeof 区别** | 数组返回整个数组大小，指针返回指针本身大小（4 或 8 字节） |
| **指针运算** | char 指针 +1 步进 1 字节，int 指针 +1 步进 4 字节 |

| 问题 | 解决方向 |
|------|----------|
| **数组退化为指针**：函数参数时丢失大小信息 | 使用模板函数保留大小；传递大小参数；使用 `std::array` 或 `std::vector` |
| **数组越界**：指针访问容易越界 | 使用数组索引；使用 STL 容器；添加边界检查 |

| 要点和思考方向 |
|----------------|
| 注意 sizeof 对指针和数组的不同结果，优先使用 STL 容器 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明数组/缓冲区大小、是否在函数间传递；如「数组作为参数传入，函数内需要知道长度」 |
| **方法** | 要求使用 `std::array`/`std::vector` 或显式传递大小；避免 `sizeof(ptr)` 误用 |
| **应用** | 将 C 风格数组改为 STL 容器；生成带边界检查的访问代码；解释 sizeof 对指针与数组的差异 |
| **提示词范例** | 「将 `void process(int* arr)` 改为使用 `std::vector` 或 `std::span`，避免丢失长度信息」 |

## 引用

| 维度 | 内容 |
|------|------|
| **作用** | 变量的别名，不占用额外内存 |
| **应用场景** | 函数参数传递（避免拷贝）；避免空指针；返回引用；范围 for 循环 |

| 问题 | 解决方向 |
|------|----------|
| **返回局部变量引用**：导致悬空引用 | 不返回局部变量引用；返回静态/全局/动态分配对象引用；使用值返回或智能指针 |
| **引用不能重新绑定**：初始化后不能改变 | 理解引用特性；需改变指向时用指针；初始化时绑定正确对象 |
| **指针引用与变量引用混淆** | 明确引用类型；指针引用是引用的指针；使用类型别名提高可读性 |

| 要点和思考方向 |
|----------------|
| 引用初始化后不能改变，不要返回局部变量引用；理解引用与指针的区别 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明参数是输入/输出、是否可选、生命周期；如「函数需要修改调用方的对象」「返回的是成员引用」 |
| **方法** | 区分值传递、引用传递、const 引用；要求 AI 检查返回引用是否悬空 |
| **应用** | 将指针参数改为引用；检查返回局部变量引用；生成范围 for 的正确用法 |
| **提示词范例** | 「检查这段代码中是否有返回局部变量引用的问题，并给出修复方案」 |

## 字符串

| 维度 | 内容 |
|------|------|
| **作用** | 存储和操作文本数据 |
| **类型** | C 风格（`char*`/`char[]`，以 `\0` 结尾）；C++（`std::string`，自动管理内存） |

| 问题 | 解决方向 |
|------|----------|
| **长度问题**：长度计算和边界处理易出错 | 使用 `std::string`；用 `size()`/`length()` 获取长度；注意 `strlen()` 不含 `\0`、`sizeof()` 含 |
| **字符 vs 字符串**：`"a"` 与 `'a'` 混淆 | `'a'` 是 char（1 字节）；`"a"` 是 const char*（2 字节含 `\0`） |
| **字符串越界**：C 风格易缓冲区溢出 | 使用 `std::string`；使用 `strncpy`、`strncat` 等安全函数；添加边界检查 |
| **拼接性能**：频繁拼接影响性能 | 使用 `append()` 或 `+=`；复杂拼接用 `ostringstream`；预分配 `reserve()` |

| 要点和思考方向 |
|----------------|
| 优先使用 `std::string`，注意字符与字符串区别，理解 C 风格以 `\0` 结尾 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明字符串来源（用户输入/配置文件/网络）、是否需要多语言、性能要求 |
| **方法** | 要求优先 `std::string`；涉及 C 接口时说明边界；要求 `reserve` 预分配 |
| **应用** | 将 C 风格字符串改为 `std::string`；生成安全拼接代码；检查缓冲区溢出风险 |
| **提示词范例** | 「将这段使用 `char*` 和 `strcpy` 的代码改为 `std::string`，并处理可能的越界」 |

## 智能指针

| 维度 | 内容 |
|------|------|
| **作用** | 解决内存泄漏，自动管理对象生命周期 |
| **应用场景** | 自动管理生命周期；避免 new/delete 泄漏；RAII；异常安全 |
| **类型** | unique_ptr（独占，不能复制只能移动）；shared_ptr（共享，引用计数）；weak_ptr（弱引用，打破循环引用） |
| **内部实现** | 重载 `*` 和 `->`；引用计数或所有权转移；超出范围自动释放 |

| 问题 | 解决方向 |
|------|----------|
| **循环引用**：shared_ptr 循环引用导致泄漏 | 使用 weak_ptr 打破；重新设计对象关系；优先用 unique_ptr |
| **性能开销**：引用计数等有开销 | 性能关键路径考虑原始指针；优先 unique_ptr；避免不必要的拷贝 |
| **原始指针混用**：所有权混乱 | 明确所有权；避免在智能指针管理对象上用原始指针；`get()` 使用要小心 |

| 要点和思考方向 |
|----------------|
| 优先使用智能指针，注意循环引用用 weak_ptr，避免与原始指针混用 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明所有权（独占/共享）、是否有循环引用风险、是否需要弱引用观察 |
| **方法** | 明确 unique_ptr/shared_ptr/weak_ptr 选择；要求检查循环引用；避免 `get()` 长期持有 |
| **应用** | 将 new/delete 改为智能指针；打破 shared_ptr 循环引用；生成 RAII 类 |
| **提示词范例** | 「这两个类互相持有 shared_ptr，可能导致泄漏，请用 weak_ptr 打破循环引用」 |

## STL（标准模板库）

| 维度 | 内容 |
|------|------|
| **作用** | 提供常用数据结构和算法，提高开发效率 |
| **应用场景** | 事件系统；对象管理；数据处理；算法实现 |

| 容器 | 特点 |
|------|------|
| vector | 动态数组，随机访问 |
| list | 双向链表，高效插入删除 |
| deque | 双端队列 |
| queue / stack | 队列 FIFO / 栈 LIFO |
| map | 红黑树，有序 |
| unordered_map | 哈希表，O(1) 查找 |
| set / unordered_set | 有序/无序集合 |

| 问题 | 解决方向 |
|------|----------|
| **性能问题**：动态扩容导致分配和碎片 | 初始化时 `reserve()` 预分配；选合适容器；用 `emplace` 替代 `push` |
| **迭代器失效**：修改容器时迭代器可能失效 | 理解失效规则；修改后重新获取；优先用索引 |
| **容器选择不当** | vector 适合随机访问，list 适合频繁插入删除；map 有序查找，unordered_map 快速查找 |

| 要点和思考方向 |
|----------------|
| 用 `reserve()` 预分配；理解迭代器失效规则；使用 emplace、范围 for 等现代特性 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明访问模式（随机/顺序、插入删除频率）、预估大小、是否需要有序 |
| **方法** | 要求选合适容器（vector/map/unordered_map 等）；要求 `reserve`/`emplace`；说明迭代器失效场景 |
| **应用** | 推荐容器类型；将裸数组改为 STL；生成带 reserve 的初始化代码；检查迭代器失效 |
| **提示词范例** | 「需要存储 10000 个玩家 ID 到属性的映射，主要操作为查找和插入，请推荐容器并给出示例代码」 |

| **应用** | 将重复代码抽象为模板；添加类型约束；解释模板编译错误 |
| **提示词范例** | 「为 mesh 顶点数据写一个泛型处理函数，支持 float3 和 double3，用 C++17 实现」 |

## Bit masking

| 维度 | 内容 |
|------|------|
| **作用** | 用位操作高效管理多个布尔状态，"多快好省" |
| **应用场景** | buff/debuff 状态；标志位；权限管理；性能优化 |

| 问题 | 解决方向 |

| **类型不安全**：位操作易出错 | 使用 `std::bitset`；强类型枚举（enum class）；封装为函数 |

| 要点和思考方向 |
|----------------|
| 用枚举提高可读性；考虑 `std::bitset`；注意优先级，用括号明确 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明状态数量、是否需要组合、可读性要求 |
| **方法** | 要求用 enum/常量定义掩码；用括号明确优先级；考虑 `std::bitset` |
| **应用** | 生成 buff/debuff 状态位定义；将多个 bool 合并为位掩码；添加注释说明含义 |
| **提示词范例** | 「用位掩码实现玩家状态（无敌、隐身、沉默），定义枚举常量并实现添加/移除/检查函数」 |

## cast

## C++17 游戏开发常用特性

> C++17 是游戏行业从 C++11/14 向现代 C++ 过渡的关键版本，编译器支持成熟，可放心全面使用。

| 特性 | 说明 | 游戏场景 |
|------|------|----------|
| **Structured Bindings** | `auto [x, y, z] = pos;` 解包 tuple/pair/struct | 解包坐标 `auto [x,y,z] = entity.GetPosition()`；遍历 map 时 `for (auto& [id, comp] : components)` 大幅提升可读性 |
| **`if constexpr`** | 编译期条件分支，不满足的分支不参与编译 | 模板化渲染管线：`if constexpr (IsDeferred) { ... } else { ... }` 一份代码编译出 Forward/Deferred 两版；消除 SFINAE 模板 |
| **`std::optional`** | 值语义的「可能不存在」，替代返回空指针 | 查询组件 `std::optional<Transform*> FindTransform(EntityID)`；射线检测 `std::optional<HitResult> Raycast(ray)`；配置表可选字段 |
| **`std::string_view`** | 零拷贝字符串视图，不拥有数据 | 配置文件解析（无需拷贝整个 value）；网络包解析（指向接收缓冲区的视图）；资源路径常量池 |
| **折叠表达式 (Fold Expressions)** | `(args + ...)` 可变参数模板展开 | 可变参数日志 `Log(severity, args...);`；组件类型列表 `RegisterComponents<Mesh, Physics, AI>();`；多通道渲染命令合并 |
| **`[[nodiscard]]`** | 属性标记，忽略返回值编译器警告 | 错误码类 `[[nodiscard]] ErrorCode DoSomething()` 防止遗漏检查；资源加载 `[[nodiscard]] ResourceHandle Load(id)` |
| **`std::variant`** | 类型安全的 union（C++17 成熟） | 事件系统数据载荷 `std::variant<DamageEvent, HealEvent, MoveEvent>`；多类型属性值存储；替代 `void*` + tag 的旧模式 |
| **`std::clamp`** | 一行代码完成值域裁剪 | 角色速度限制、UI 坐标裁剪、颜色分量规范化：`r = std::clamp(r, 0.0f, 1.0f)` |
| **`std::filesystem`** | 跨平台文件系统操作 | 资源目录遍历、打包工具、热更文件监控；替代 `#ifdef _WIN32` 的分平台路径拼接 |

## C++20 游戏革命

> C++20 是 C++ 游戏开发的「分水岭」——协程、Concepts、Modules 三项特性从根本改变引擎架构方式。Unreal Engine 5 已大量采用 C++20。

| 特性 | 说明 | 游戏场景 |
|------|------|----------|
| **Concepts** | 模板参数的类型约束，编译期检查接口 | 约束渲染对象 `template<Renderable T> void Draw(T& obj)`；UE5 已采用 `CStaticClassProvider` 等 Concept 约束反射系统；让模板错误信息清晰 100 倍 |
| **Coroutines** | 无栈协程，`co_await`/`co_yield` | **异步资源加载**：`co_await AssetManager::LoadAsync(path)` 避免回调地狱；**分帧计算**：`co_yield` 将大量计算分散到多帧；网络请求流程化（登录→验证→进游戏） |
| **Ranges** | 管道式数据处理，`\|` 连接操作 | 剔除不可见对象 `entities \| filter(IsVisible) \| transform(GetBounds) \| to<vector>()`；伤害计算管道；粒子系统批量更新 |
| **Modules** | 替代 `#include`，模块化编译 | 编译速度提升 3-10 倍（UE5 5.4+ 已支持模块）；引擎模块解耦（Core/Physics/Audio 独立模块）；消除头文件宏污染 |
| **`std::span`** | 无所有权数组视图，统一数组+vector+原始指针接口 | 顶点缓冲 `void ProcessVertices(std::span<Vertex> verts)` 统一接受 `std::vector`/`std::array`/C 数组；避免指针+长度双参数 |
| **`constexpr` vector/string** | 编译期动态容器 | 编译期生成查找表（LUT）、三角函数表、CRC 表；字符串哈希在编译期完成 `constexpr auto hash = "path/to/asset"_hash` |
| **三路比较 `<=>`** | 自动生成所有比较运算符 | `auto operator<=>(const Config&) const = default;` 一行代码完成排序、查找支持；版本号比较、资源 ID 排序 |
| **`std::format`** | 类型安全的字符串格式化 | 替代 `sprintf`/`stringstream`：`std::format("Frame {}: {} entities, {:.2f} ms", frame, count, dt)`；日志系统格式化升级 |

## C++23 游戏开发前瞻

> C++23 处于「早期采用」阶段，以下特性值得关注并在工具链代码中逐步引入。

| 特性 | 说明 | 游戏场景前瞻 |
|------|------|-------------|
| **`std::expected`** | 类似 Rust 的 `Result<T, E>`，返回值或错误 | 替代 `(bool success, T value)` 模式：`std::expected<MeshData, LoadError> LoadMesh(path)`；解析错误链式传播 |
| **`std::mdspan`** | 多维数组视图，不拥有数据 | 纹理数据 `mdspan<uint8_t, dextents<2, 1024, 1024>> pixels`；矩阵运算的多维视图；骨骼动画矩阵批量访问 |

| **`std::flat_map`** | 排序 vector 实现的 map，缓存友好 | 小型配置表（< 100 条）查询比 `std::map` 快 3-5 倍；热路径属性查找 |

## 现代 C++ 游戏开发最佳实践

> 以下对照表总结了从「经典 C++」到「现代 C++（C++17/20）」的核心迁移。游戏引擎代码中对性能敏感的路径需谨慎评估，但工具链和上层逻辑应全面采用现代写法。

| 旧做法 | 现代做法 | 说明 |
|--------|----------|------|
| `new` / `delete` 裸指针 | `std::unique_ptr` / `std::shared_ptr` | 明确所有权，RAII 自动释放；性能敏感热路径可用原始指针（无所有权语义）标记意图 |
| 运行时多态 + `dynamic_cast` | `std::variant` + `std::visit` | 类型安全、无虚函数开销；适合封闭类型集（如事件类型、命令类型） |
| 回调函数指针 / `std::function` 嵌套 | C++20 Coroutines (`co_await`) | 异步逻辑线性化，可读性大幅提升；异步加载、分帧计算的主要重构方向 |
| `#define` 常量 / 宏函数 | `constexpr` / `consteval` / `enum class` | 类型安全、作用域控制、可调试；`enum class` 避免全局命名污染 |
| `std::pair` / `std::tuple` 成员访问 | C++17 Structured Bindings | `auto [name, score] = player` 语义清晰，消除 `.first`/`.second` 困惑 |
| `(T* ptr, size_t count)` 参数对 | `std::span<T>` | 统一接口，自动推导大小（数组），安全访问 |
| 返回 `bool` + 引用输出参数 | `std::optional` / `std::expected` | 意图明确，消除未初始化变量的风险，链式处理 |
| SFINAE 模板约束 | C++20 Concepts | 错误信息清晰 100 倍，IDE 智能提示准确，接口契约显式化 |
| C 风格 `(Type)value` 转换 | `static_cast` / `narrow_cast` | 可见性高、grep 友好、编译器可检查；`narrow_cast` 显式标记窄化 |
| `#include` 头文件 | C++20 Modules | 编译速度 3-10 倍提升，消除宏污染，模块接口显式化 |
| 手动 for 循环 + 临时变量 | C++20 Ranges `\|` 管道 | 声明式、可组合、惰性求值；`entities \| filter(active) \| transform(pos) \| to<vector>()` |
| `union` + tag 判断类型 | `std::variant` + `std::visit` | 类型安全，编译器保证访问正确性，不会访问错误成员 |
| `va_list` / `printf` 风格日志 | `std::format` / `std::print` (C++20/23) | 类型安全，编译期验证格式串，支持自定义类型格式化 |
