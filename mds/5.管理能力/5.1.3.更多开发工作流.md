## 更多开发工作流
<p>

</p>

## 工作流概览

## 数据配置更新

| 维度 | 内容 |
|------|------|
| **作用** | 游戏配置数据的制作、更新和管理流程 |
| **应用场景** | 所有需要数据驱动的游戏功能<br>角色属性配置<br>道具数据配置<br>关卡配置<br>数值平衡配置 |
| **做什么的？** | 游戏配置数据的制作、更新和管理流程 |
| **在哪用？** | 所有需要数据驱动的游戏功能，如角色属性、道具数据、关卡配置等 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现数据驱动开发？** | 数据驱动开发：通过配置数据而非硬编码来实现游戏功能，提高灵活性和可维护性<br>使用配置数据<br>避免硬编码<br>提高灵活性<br>提高可维护性<br>设计数据表结构<br>实现数据加载系统<br>实现数据驱动逻辑 |
| **如何管理配置数据的版本？** | 版本化管理：配置数据需要版本管理，支持回滚和对比<br>实现版本管理<br>支持版本回滚<br>支持版本对比<br>实现版本管理系统<br>实现版本回滚机制<br>实现版本对比工具 |
| **如何确保前后端数据同步？** | 前后端同步获取数据：确保前后端使用相同的数据结构，避免不一致<br>使用相同数据结构<br>实现数据同步机制<br>避免数据不一致<br>建立数据结构规范<br>实现数据同步机制<br>实现数据一致性检查 |
| **如何实现数据的实时生效？** | 实时生效（可测试）：配置数据更新后能够实时生效，方便测试和调试<br>实现数据热更新<br>实现实时生效机制<br>方便测试和调试<br>实现热更新机制<br>实现实时生效系统<br>优化更新性能 |
| **如何检查数据的合法性？** | 数据合法性检查：对配置数据进行校验，避免错误数据导致游戏问题<br>实现数据校验<br>避免错误数据<br>保证数据质量<br>实现数据校验规则<br>实现数据校验工具<br>优化校验性能 |
| **如何实现协作流程？** | 协作流程：<br>1. 数据表设计：策划/程序 — 程序和策划根据功能需求共同敲定数据表结构<br>2. 数据表填写：策划/运营 — 更新游戏数据，填写配置内容<br>3. 数据集成：程序 — 最好由自动化的工具来完成，将配置数据集成到游戏中<br>建立协作流程<br>实现自动化工具<br>优化协作效率 |

| 要点和思考方向 |
|----------------|
| 数据配置更新是数据驱动开发的基础 |
| 实现数据驱动开发提高灵活性和可维护性 |
| 管理配置数据版本支持回滚和对比 |
| 确保前后端数据同步避免不一致 |
| 实现数据实时生效方便测试和调试 |
| 检查数据合法性保证数据质量 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「数据驱动」「配表」「版本管理」「前后端同步」「热更新」「数据校验」 |
| **方法** | 让 AI 设计表结构、协作流程、校验规则、集成脚本 |
| **应用** | 配表设计、数据校验、热更新、前后端同步 |
| **提示词范例** | 「游戏需数据配置流程：策划配表、程序集成、前后端共用结构，请设计 Excel 导出与校验规则，以及热更新与版本回滚的实现要点」 |

## （文本）本地化

| 维度 | 内容 |
|------|------|
| **作用** | 游戏文本的多语言本地化流程 |
| **应用场景** | 需要支持多语言的游戏<br>国际化游戏<br>多地区运营 |
| **做什么的？** | 游戏文本的多语言本地化流程 |
| **在哪用？** | 需要支持多语言的游戏 |

| 问题 | 解决方向 |
|------|----------|
| **如何收集和管理本地化文本Key？** | 协作流程：<br>1. 收集文本Key：策划/程序 — 收集Key，并填写到本地化文本表中，需要确保Key的唯一性和可读性<br>建立Key命名规范<br>实现Key管理系统<br>实现Key唯一性检查 |
| **如何支持通过Excel生成本地化文本？** | 支持通过Excel生成本地化文本：使用Excel等工具管理本地化文本，方便翻译团队协作<br>实现Excel导入导出<br>实现文本格式转换<br>优化工具使用 |
| **如何高效地完成翻译和集成？** | 协作流程：<br>2. 本地化文本：翻译团队 — 制作/更新本地化文本表，进行多语言翻译<br>3. 导出/集成到游戏中：程序 — 最好由自动化的工具来完成，将本地化文本导出并集成到游戏中<br>建立翻译流程<br>实现自动化工具<br>优化集成效率 |

| 要点和思考方向 |
|----------------|
| 本地化是国际化游戏的基础 |
| 收集和管理本地化文本Key确保唯一性和可读性 |
| 支持通过Excel生成本地化文本方便翻译协作 |
| 实现自动化工具提高翻译和集成效率 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「本地化」「多语言」「Key」「Excel」「翻译」「导出」 |
| **方法** | 让 AI 设计 Key 规范、Excel 模板、导出与集成流程 |
| **应用** | Key 管理、翻译表、多语言集成 |
| **提示词范例** | 「游戏需本地化：策划收集 Key、翻译填 Excel、程序导出集成，请设计 Key 命名规范、Excel 表结构，以及导出为游戏可用的自动化脚本」 |

## 技能

| 维度 | 内容 |
|------|------|
| **作用** | 游戏技能系统的制作流程，包括设计、特效、音效、配表和集成 |
| **应用场景** | 有技能系统的游戏<br>RPG游戏<br>MOBA游戏<br>动作游戏 |
| **做什么的？** | 游戏技能系统的制作流程，包括设计、特效、音效、配表和集成 |
| **在哪用？** | 有技能系统的游戏，如RPG、MOBA、动作游戏等 |

| 问题 | 解决方向 |
|------|----------|
| **如何协调技能的设计、特效、音效和配置？** | 协作流程：<br>设计：策划设计技能的功能、效果、数值等<br>特效/动画：动画师 — 借助动画/粒子等实现技能相关的视觉效果（释放效果、击中效果、UI效果、角色动画）<br>音效：音效师 — 配合特效制作音效，增强技能的听觉体验<br>配表：策划 — 配置技能的功能和表现，包括伤害、范围、冷却时间等<br>建立协作流程<br>协调制作进度<br>确保内容一致 |
| **如何确保技能的表现和逻辑一致？** | 协作流程：<br>集成：程序 — 集成效果和配置，实现技能的逻辑，确保技能的表现和功能一致<br>实现技能系统<br>实现表现逻辑对接<br>确保一致性 |

| 要点和思考方向 |
|----------------|
| 技能制作需要多角色协作 |
| 协调技能的设计、特效、音效和配置 |
| 确保技能的表现和逻辑一致 |
| 建立清晰的协作流程 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「技能」「特效」「音效」「配表」「释放」「击中」「集成」 |
| **方法** | 让 AI 设计技能配表结构、协作流程、表现与逻辑对接 |
| **应用** | 技能配表、特效配置、多角色协作流程 |
| **提示词范例** | 「技能系统需策划设计、美术特效/音效、程序集成，请设计技能配表结构（伤害、范围、冷却、特效引用），以及策划/美术/程序协作流程与交付物规范」 |

## 版本制作

| 维度 | 内容 |
|------|------|
| **作用** | 游戏版本的构建、测试和发布流程 |
| **应用场景** | 所有需要测试和发布的游戏版本<br>测试版本制作<br>正式版本发布 |
| **做什么的？** | 游戏版本的构建、测试和发布流程 |
| **在哪用？** | 所有需要测试和发布的游戏版本 |

| 问题 | 解决方向 |
|------|----------|
| **如何高效地制作测试版本？** | 测试：为QA提供可测试的版本<br>制作测试版本<br>提供给QA测试<br>实现自动化构建<br>优化构建流程<br>提高制作效率 |
| **如何自动化版本构建和发布？** | 发布：制作正式发布版本<br>出包工作流：<br> - 人工：手动执行构建、打包、签名等步骤，适合初期或特殊版本<br> - 自动：通过自动化工具链完成版本制作，提高效率和一致性（参考 [4.3.3.DevOps](../4.生产能力/4.3.3.DevOps.md) 中的持续集成和持续交付相关内容）<br>实现自动化工具链<br>优化自动化流程<br>提高效率和一致性 |

| 要点和思考方向 |
|----------------|
| 版本制作是游戏发布的基础 |
| 高效制作测试版本为QA提供支持 |
| 实现自动化版本构建和发布提高效率 |
| 参考DevOps实践优化版本制作流程 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「出包」「构建」「测试版本」「发布」「自动化」「Jenkins」 |
| **方法** | 让 AI 设计构建流程、出包流水线、与 DevOps 的衔接 |
| **应用** | 测试版本制作、发布流程、自动化出包 |
| **提示词范例** | 「需版本制作流程：测试版供 QA、正式版发布，请设计 Jenkins 或类似工具的流水线，以及人工出包与自动出包的切换策略」 |
